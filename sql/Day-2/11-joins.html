üìò Definition
JOIN combines rows from two or more tables based on a related column.
üí° Why Use JOIN?

Our tables are related:

  • students table has student information
  • enrollments table links students to courses
  • courses table has course information

JOIN helps us see all this information together!

1. INNER JOIN (Most Common)
üëâ Meaning
Returns only matching rows from both tables
üß† Example Code 
-- Show student names with their enrolled courses
SELECT s.name, c.course_name
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id;
üìù Result: Shows which student is enrolled in which course
Only shows students who have enrollments
üîç How JOIN Works 
-- Step 1: Start with enrollments table
-- Step 2: JOIN students table using student_id
-- Step 3: JOIN courses table using course_id
-- Result: We get student name + course name together

SELECT 
    s.name AS student_name,
    c.course_name,
    e.enroll_date
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id;
2. Simple Two-Table JOIN
üß† Example Code 
-- Students with their enrollment dates
SELECT s.name, e.enroll_date
FROM enrollments e
JOIN students s ON e.student_id = s.student_id;
3. JOIN with WHERE Clause
üß† Example Code 
-- Active students and their courses
SELECT s.name, c.course_name
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id
WHERE s.status = 'active';
4. JOIN with Aggregate Functions
üß† Example Code 
-- Count courses per student
SELECT s.name, COUNT(c.course_id) AS course_count
FROM students s
JOIN enrollments e ON s.student_id = e.student_id
JOIN courses c ON e.course_id = c.course_id
GROUP BY s.name;
5. LEFT JOIN (Optional)
üëâ Meaning
Returns all rows from left table, even if no match in right table
üß† Example Code 
-- Show ALL students, even if they have no enrollment
SELECT s.name, c.course_name
FROM students s
LEFT JOIN enrollments e ON s.student_id = e.student_id
LEFT JOIN courses c ON e.course_id = c.course_id;
üìù Result: Shows all students, course_name will be NULL for students with no enrollments
‚úÖ Complete Real-Life Examples 
-- Example 1: Student names with course names and fees
SELECT 
    s.name AS student_name,
    c.course_name,
    c.fees
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id;

-- Example 2: Total fees per student
SELECT 
    s.name,
    SUM(c.fees) AS total_fees
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id
GROUP BY s.name;

-- Example 3: Courses enrolled by students from Delhi
SELECT 
    s.name,
    c.course_name
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id
WHERE s.city = 'Delhi';
üìä JOIN Types Summary
  • INNER JOIN (or just JOIN): Only matching rows
  • LEFT JOIN: All rows from left table + matching from right
  • RIGHT JOIN: All rows from right table + matching from left
  • FULL JOIN: All rows from both tables

For beginners, focus on INNER JOIN first!
