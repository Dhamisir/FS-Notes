In React, if you use a standard <a href="..."> tag, the entire page will reload. This kills the performance benefit of using React!

‚ùå The Wrong Way (Standard HTML)

Using <a href="/login">Login</a> causes a full reload. You will lose your app's state!

‚úÖ The React Way (Link Component)

Instead, we use the <Link> component provided by react-router-dom.

import { Link } from "react-router-dom";

<Link to="/login">Login</Link>

üß© Practical Example: A Navbar

You can create a Navbar component that appears on every page.

import { Link } from "react-router-dom";

function Navbar() {
  return (
    <nav>
      <Link to="/">Home</Link> |
      <Link to="/login">Login</Link> |
      <Link to="/about">About</Link>
    </nav>
  );
}

üß† Why use Link?

  • It changes the URL in the address bar.
  • It tells React to "swap" the component.
  • It does NOT refresh the browser tab.
