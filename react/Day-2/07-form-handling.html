üìù Form Handling (Controlled Components)

In standard HTML, the input box "holds" its own value. In React, we want React to control everything.

üß† What is a Controlled Component?

It means the value of the input is synced with a State variable. Every time the user types, the state updates, and the input shows the state value.

üß© Single Input Example

function NameForm() {
  const [name, setName] = useState("");

  return (
    <div>
      <input
        type="text"
        value={name} {/* Sync state to input */}
        onChange={(e) => setName(e.target.value)} {/* Sync input to state */}
      />
      <p>Preview Name: {name}</p>
    </div>
  );
}

üöÄ Handling Form Submission

When the user clicks "Submit", we need to prevent the page from refreshing üîÑ.

const handleSubmit = (e) => {
  e.preventDefault(); // üõë Very important! Stops page reload
  alert("Form submitted with name: " + name);
};

<form onSubmit={handleSubmit}>
  <input ... />
  <button type="submit">Submit</button>
</form>

üîë Why use e.preventDefault()?

By default, when you submit a form, the browser tries to send it to the server and reloads the page. We want React to handle it instead, so we use this function to stop the default browser behavior.
