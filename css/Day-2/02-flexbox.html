Step 1: The "Normal" Document Flow

By default, div elements are block level. They stack on top of each other and take 100% width.

Box 3
Observation: Boxes are vertical. They don't sit side-by-side even if there is space.

Step 2: Add display: flex

The moment you add this to the PARENT, all children become "flex items".

After display: flex;
Box 1
Box 2
Box 3
.container { display: flex; }
Observation: Magic! Boxes immediately jump to a row. They now only take as much width as their content needs.

Step 3: Horizontal Control (The Main Axis)

Now that they are in a row, how do we move them left, right, or center? We use justify-content.

justify-content: flex-start; /* Default (Left) */ justify-content: center; /* Perfect Center */ justify-content: flex-end; /* Right */ justify-content: space-between; /* Push boxes to edges */ justify-content: space-around; /* Even space around boxes */

Step 4: Vertical Control (The Cross Axis)

How do we move boxes up and down? We use align-items.

With align-items: center; (Container height is 150px)
Perfectly Centered Vertically
align-items: flex-start; /* Top */ align-items: center; /* Middle */ align-items: flex-end; /* Bottom */ align-items: stretch; /* Default (take full height) */

Step 5: The Direction Swap

What if we want them back in a column but still want to use Flex powers? flex-direction.

With flex-direction: column;
Row is now Column
Alignment now works Vertically
CRITICAL: When you switch to column, justify-content now controls top/bottom, and align-items now controls left/right. The axes swap!

Step 6: Don't Squash! (flex-wrap)

By default, Flex tries to force everything into one line. If you have 50 boxes, they will shrink until they break. flex-wrap fixes this.

flex-wrap: wrap; /* Let items jump to the next line */ gap: 15px; /* The "Pro" way to add space between wrapped items */
