Modern websites must work on everything from a huge 4K monitor to a tiny smartphone. We use Media Queries to change styles based on the screen size.

1. The "Mobile First" Philisophy

Instead of designing for Desktop and "fixing" for Mobile, experts do the opposite:

  1. Write your normal CSS for Mobile (Small screens).
  2. Use Media Queries to add complexity as the screen gets Larger.

2. Basic Media Query Syntax

/* This style applies to ALL screens */ .container { width: 100%; padding: 20px; } /* This only applies when screen is 768px or LARGER (Tablets/Laptops) */ @media (min-width: 768px) { .container { width: 80%; margin: 0 auto; } } /* This only applies when screen is 1200px or LARGER (Desktop) */ @media (min-width: 1200px) { .container { width: 1140px; } }

3. The Viewport Meta Tag

Media queries will NOT work on mobile devices unless you have this tag in your HTML <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

4. Testing Room

Common Breakpoints:

  • 576px: Small (Mobile Landscape)
  • 768px: Medium (Tablets)
  • 992px: Large (Laptops)
  • 1200px: Extra Large (Desktop)
