üé® Types of CSS & Their Evolution

CSS (Cascading Style Sheets) can be added to an HTML document in three ways. Each method was an improvement over the previous one to solve specific problems in web development.

1Ô∏è‚É£ Inline CSS

Style is applied directly inside the HTML tag using the style attribute.

<h1 style="color: blue; text-align: center;">Hello World</h1>
<p style="color: red; font-size: 20px;">This is a paragraph.</p>

üö® Problem with Inline CSS:

  • Repetition: If you have 10 paragraphs and want them all red, you must write the style 10 times.
  • Messy HTML: Logic and styling are mixed together, making the code hard to read.
  • Hard to Maintain: If you want to change the color from red to green, you must update 10 different tags.

2Ô∏è‚É£ Internal (Embedded) CSS

‚úÖ Solution: Internal CSS

Internal CSS solves the repetition problem by moving styles to a <style> tag inside the <head> section.

<head>
  <style>
    p {
      color: red;
      font-size: 20px;
    }
  </style>
</head>
<body>
  <p>Paragraph 1</p>
  <p>Paragraph 2</p>
</body>

üö® Problem with Internal CSS:

  • Limited Scope: Styles only apply to that specific page. If your website has 50 pages, you must copy-paste the <style> tag into 50 files.
  • Large File Size: HTML files become very heavy as styles grow.

3Ô∏è‚É£ External CSS

‚úÖ Final Solution: External CSS

This is the professional way. We create a separate .css file and link it to our HTML pages using the <link> tag. This solves the "Multiple Page" problem.

Step 1: Create style.css 
/* style.css */
p {
  color: red;
  font-size: 20px;
}
Step 2: Link in HTML 
<head>
  <link rel="stylesheet" href="style.css">
</head>

üí™ Why External is Best:

  • Change Once, Apply Everywhere: Update the CSS file, and all 50 pages change instantly.
  • Clean Code: HTML is for structure, CSS is for look. Separated logic.
  • Caching: Browsers download the CSS file once and reuse it for other pages, making the site faster.
